Digital Oceans

SERVIDORES CLOUD

servidores vituales privados.

1. Inicie sesión en su cuenta de DigitalOcean
Crear una cuenta en https://cloud.digitalocean.com/registrations/new
2. Hacer clic en crear droplets
En digital Oceans una gota o droplets equivale a un servidor
3. En Elija una imagen
Permite seleccionar :
1-seleccionar una distribución de so Linux
2-se puede instalar una distribución de Linux con una aplicación
3-o instalar una imagen basado en uno de tus respaldos
4. En Elegir un tamaño , seleccione la especificación de su máquina
Selecciona el tamaño del servidor dependiendo del proyecto, ademas permite la opcion de uso por hora
5. En Elegir una región de centro de datos , seleccione su región.
Seleccionar donde se ubicara el servidor
6. Agregue una clave SSH.
Para nuevas llaves
1. En Agregar sus claves SSH, haga clic en Nueva clave SSH (esto abre un cuadro de diálogo) .
2. Desde su terminal, ejecute cat $HOME/.ssh/id_rsa.pub.
3. Copie el resultado y péguelo en el cuadro de diálogo Nueva clave SSH , proporcione un nombre y haga clic en Agregar clave SSH .
Para llaves existentes
Simplemente agregue una marca de verificación junto a las claves existentes que desea agregar.
7. En Finalizar y crear , asigne a su Droplet un nombre de host (no obligatorio) y haga clic en Crear .
...
8. Una vez creado, copie la dirección IP en su portapapeles.
...
9. En un navegador, vaya a la dirección IP que copió anteriormente y complete el formulario presentado para completar la configuración. De lo contrario, puede permitir que otros reconfiguren el acceso SSH en su servidor.
...
10. Ahora accedemos a nuestro servidor para prepararlo a la aplicación que recibirá después
a) Usamos el comando
Ssh root@nuestraIp
b) Cuando entremos en el servidor usaremos el comando
Dokku apps:create nombreDeLaAppicacion
d) Procederemos a instalar el plugins de la base de datos
Sudo dokku plugin:install http:/…Direccion donde se encuentra el plugins
e) Luego crearemos el servicio de la base de datos
Dokku gestorDeBaseDeDatos:créate nombreDeLaApps-database
f) Ahora queremos unir la base de datos con la aplicación
Dokku gestorDeBaseDeDatos:link nombreDelServicio nombreDeLaApp
g) Salimos del Servidor
exit logout
closed
11. Ultimos pasos
a) Colocamos el proyecto en una carpeta
b) Entramos al directorio donde se encuentra nuestra aplicación
c) Instalamos el remoto para subir la aplicación
d) Para instalar la aplicación corremos el comando git push doku master
e) Allí se muestra el ip y el puerto
f) Ahora accedemos al navegador con la url que se muestra anteriormente
