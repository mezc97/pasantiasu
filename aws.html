Amazon Web Server

SERVIDORES CLOUD

servidores vituales privados.

1. Inicie sesión en su cuenta en Amazon Web Services
Crear una cuenta en https://azure.microsoft.com/es-es/
2. Instalación
a) Clonar el repo Dokku a su máquina local
git clone git@github.com:dokku/dokku.git
b) Instalar Vagrant
brew cask install vagrant
c) Instale VirtualBox para configurar Vagrant
brew cask install virtualbox
3. Configurar proyecto para Dokku
Configure Dokku en su máquina Vagrant local:
a) Cree la máquina virtual Vagrant cambiando el directorio a su repositorio local de Dokku
cd /path/to/local/dokku
b) Crea una máquina virtual vagabunda con Dokku
vagrant up
c) Desea establecer la IP local (el valor predeterminado es 10.0.0.2) de su máquina Vagrant en su /etc/hostsarchivo y elegir un nombre de host. Para este tutorial, usaremos el nombre de hostdokku.me
10.0.0.2 dokku.me
4. Termine de configurar Dokku en su máquina Vagrant local
a) Acceda dokku.me en su navegador.
b) Pegue su clave pública ssh, que se encuentra en ~/.ssh/id_rsa.pub. Esta clave pública ssh debe ser la misma clave que usó para clonar la aplicación de ejemplo .
c) Establecer Hostnameendokku.me
d) Marque Use virtualhost naming for appsEsto le permitirá acceder <project-name>.dokku.me
e) Edite su /etc/hostsarchivo y agregue otro host 10.0.0.2 express-example.dokku.me
f) Agregue la nueva instancia de Dokku como un git remoto a nuestra aplicación de ejemplo para que podamos comenzar a enviar cambios localmente a Dokku en nuestra máquina virtual Vagrant local git remote add local dokku@dokku.me:express-example
g) Realice un cambio en su vista en la aplicación de ejemplo Express y envíe sus cambios git push local master
5. Configurar AWS
a) Inicie una nueva instancia para EC2.
b) Seleccione cualquier tipo de instancia. Para este tutorial, probablemente desee seleccionar el t2.microtipo, ya que es elegible para usuarios de nivel gratuito.
c) Seleccione cualquier tipo de instancia. Para este tutorial, probablemente desee seleccionar el t2.microtipo, ya que es elegible para usuarios de nivel gratuito.
Configure su grupo de seguridad agregando tipos:
c.a) SSH en puerto 22
c.b) HTTP en puerto 80
c.c) También puede ser necesario agregar TCPun rango de puertos 20000-65535
d) Inicie su nueva instancia y descargue el .pemarchivo para acceso ssh.
Una vez que haya lanzado su instancia, obtenga su IP pública y agréguela a su /etc/hostsarchivo. Usaremos el nombre de hostdokku.aws.me
<instance's public IP> dokku.aws.me
6. Configure Dokku en su instancia EC2
a) SSH en su instancia EC2
ssh -i /path/to/file.pem ubuntu@dokku.aws.me
b) Instalar Dokku a través de wget
wget https://raw.githubusercontent.com/progrium/dokku/v0.4.5/bootstrap.sh
sudo DOKKU_TAG=v0.4.5 bash
c) Una vez que Dokku esté instalado, acceda a dokku.aws.me en su navegador.
d) Pegue la misma clave pública ssh que antes, desde ~/.ssh/id_rsa.pub
e) Establecer Hostnameendokku.aws.me
f) Cheque Use virtualhost naming for apps
g) Agregue otro nombre de host a su /etc/hostsarchivo.s
<instance's public IP> express-example.dokku.aws.me
7. configuración para Dokku
a) Agregue la instancia EC2 como un git remoto a nuestra aplicación de ejemplo para que podamos comenzar a enviar cambios de forma remota a Dokku en nuestro servidor EC2 Ubuntu.
git remote add aws dokku@dokku.aws.me:express-example
b) Realice un cambio en su vista en la aplicación de ejemplo Express y envíe sus cambios
git push aws master
